- `JsonRpcError` - JSON-RPC 错误
- `TransportError` - 传输层错误
- `ToolNotFound` - 工具未找到

- `Unknown` - 未知错误

**JSON-RPC 错误码**:
- `-32700`: Parse error
- `-32600`: Invalid request
- `-32601`: Method not found
- `-32602`: Invalid params
- `-32603`: Internal error

## 📝 关键决策

### 1. JSON-RPC 2.0 协议选择
**决策**: 使用 JSON-RPC 2.0 作为通信协议

**理由**:
- MCP 协议基于 JSON-RPC 2.0
- 简单易实现
- 广泛支持

### 2. 传输层抽象
**决策**: 定义 `Transport` trait

**理由**:
- 支持多种传输方式（Stdio/TCP/HTTP）
- 易于测试（Mock）
- 可扩展性

### 3. 三大核心能力
**决策**: 同时实现 Tools/Resources/Prompts

**理由**:
- MCP 协议的三大核心
- 提供完整的 MCP 支持
- 满足不同使用场景
