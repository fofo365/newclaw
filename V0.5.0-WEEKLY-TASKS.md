## 🔧 技术细节

### ContextManager 统一方案

#### 问题
- `src/context/mod.rs` 定义了 ContextManager
- `src/core/context.rs` 也定义了 ContextManager
- 导致混淆和维护困难

#### 解决方案
1. 保留 `src/core/context.rs` 的 ContextManager 作为主要实现
2. 将 `src/context/mod.rs` 改为重新导出
3. 更新所有引用

#### 实现步骤
```rust
// src/context/mod.rs
pub use core::context::ContextManager;

// 或者合并两个实现，选择最优方案
```

### 嵌入配置文件

```toml
# newclaw.toml

[context]
enabled = true
max_tokens = 8000
strategy = "semantic"

[context.embedding]
model = "text-embedding-3-small"
batch_size = 10
timeout = 30
max_retries = 3
enable_cache = true
cache_ttl = 86400  # 24 hours

[context.retrieval]
top_k = 5
similarity_threshold = 0.7
enable_rerank = false
```

### 性能基准测试

```bash
# 嵌入性能
cargo bench --bench embedding

# 缓存性能
cargo bench --bench cache

# 端到端性能
cargo bench --bench context_manager
```
