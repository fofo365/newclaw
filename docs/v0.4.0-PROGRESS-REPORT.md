    pub base_url: String,

    pub max_heartbeat_failures: u32,      // 3
    pub enable_auto_reconnect: bool,
    pub max_reconnect_attempts: u32,      // 10
    pub initial_reconnect_delay: Duration, // 1s
    pub max_reconnect_delay: Duration,     // 60s

### 4. 事件驱动架构
```rust
#[async_trait]
pub trait EventHandler: Send + Sync {
    async fn handle(&self, event: FeishuEvent) -> WebSocketResult<()>;
    async fn on_connect(&self, app_id: &str) -> WebSocketResult<()>;
    async fn on_disconnect(&self, app_id: &str) -> WebSocketResult<()>;
    async fn on_error(&self, app_id: &str, error: &WebSocketError) -> WebSocketResult<()>;
}
```

## 💡 经验总结

### 成功要素
1. **渐进式开发** - 先设计后实现，每步都有清晰目标
2. **测试驱动** - 每个组件都有完整的单元测试
3. **文档同步** - 设计文档和代码同步更新
4. **模块化设计** - 每个组件职责单一，易于测试和维护

### 技术决策
1. **使用 Arc<RwLock>** - 线程安全且支持并发读
2. **异步 Trait** - 使用 async_trait 简化异步接口
3. **枚举错误类型** - thiserror 提供清晰的错误信息
4. **配置结构体** - Default trait 提供合理默认值
