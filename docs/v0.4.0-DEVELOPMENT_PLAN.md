## 💻 技术栈

### 后端（Rust）
- **WebSocket**: tokio-tungstenite
- **HTTP**: axum
- **序列化**: serde + serde_json
- **异步运行时**: tokio
- **日志**: tracing

### 前端（Dashboard）
- **框架**: React + TypeScript
- **构建工具**: Vite
- **UI 组件库**: Ant Design / Material-UI
- **状态管理**: Zustand / Redux Toolkit
- **WebSocket**: native WebSocket API
- **HTTP**: axios

### 数据库
- **SQLite**: 本地存储
- **Redis**: 缓存和消息队列（可选）

## 🚨 风险管理

### 技术风险
1. **WebSocket 稳定性**: 可能遇到连接断开、重连失败
   - **缓解**: 实现完善的错误处理和重试机制
   
2. **Dashboard 性能**: 大量数据可能导致性能问题
   - **缓解**: 实现分页、虚拟滚动、懒加载

3. **单体间通信**: 复杂度高，可能延期
   - **缓解**: 简化设计，优先核心功能

### 时间风险
1. **任务评估不准确**: 可能低估某些任务
   - **缓解**: 预留 20% 缓冲时间

2. **外部依赖**: 可能有其他任务插入
   - **缓解**: 优先级明确，快速响应
