## 💡 技术方案

### 1. 浏览器控制 (browser)
**技术选型**: `playwright-core` crate (Rust)

**功能**:
- 页面导航、点击、输入
- 截图、PDF 生成
- 等待元素、执行 JS
- 多标签页管理

**依赖**:
```toml
playwright-core = "0.1"
tokio = { version = "1", features = ["full"] }
```

### 2. Canvas 展示 (canvas)
**技术选型**: Headless Chrome + Rust

**功能**:
- present (展示 URL/HTML)
- snapshot (截图)
- eval (执行 JS)
- navigate (导航)

**依赖**:
```toml
headless_chrome = "1.0"
image = "0.24"
```

### 3. 节点管理 (nodes)
**技术选型**: HTTP API + WebSocket

**功能**:
- 节点发现和配对
- 通知发送
- 相机/照片控制
- 屏幕录制
- 位置获取
- 命令执行

**依赖**:
```toml
reqwest = { version = "0.11", features = ["json", "rustls-tls"] }
tokio-tungstenite = "0.20"
```

### 4. 会话管理 (sessions)
**技术选型**: 内置会话管理器

**功能**:
- spawn (创建子会话)
- send (发送消息)
- list (列出会话)
- history (历史记录)

**实现**:
```rust
pub struct SessionManager {
    sessions: HashMap<String, Session>,
    history: VecDeque<Message>,
}
```

### 5. 记忆系统 (memory)
**技术选型**: 向量数据库 (Qdrant/Meilisearch)

**功能**:
- search (语义搜索)
- get (获取记忆片段)
- 向量嵌入集成

**依赖**:
```toml
qdrant-client = "1.5"
tiktoken-rs = "0.5"
```

### 6. 子代理 (subagents)
**技术选型**: 进程管理 + 通信协议

**功能**:
- list (列出子代理)
- steer (引导子代理)
- kill (终止子代理)

**实现**:
```rust
pub struct SubagentManager {
    processes: HashMap<String, Child>,
    channels: HashMap<String, Sender>,
}
```

## 🚨 风险评估

### 高风险
1. **浏览器控制**: Playwright Rust crate 可能不成熟
   - **缓解**: 先用 headless Chrome，后续迁移到 Playwright

2. **节点管理**: 依赖外部服务 (OpenClaw Gateway)
   - **缓解**: 实现独立的节点管理协议

### 中风险
1. **Canvas 展示**: Headless Chrome 资源消耗大
   - **缓解**: 限制并发实例数，添加超时控制

2. **记忆系统**: 向量数据库部署复杂
   - **缓解**: 先用内存向量存储，后续支持 Qdrant

### 低风险
1. **会话管理**: 实现相对简单
2. **子代理**: 基于进程管理，技术成熟
