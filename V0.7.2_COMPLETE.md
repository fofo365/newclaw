# NewClaw v0.7.2 最终修复总结

## 修复内容

### 1. 飞书令牌类型混淆 (e9de13fe)
- 区分 `tenant_access_token` (t-) 和 `user_access_token` (u-)
- 创建/更新文档强制使用 user_access_token
- 添加令牌格式验证（后临时移除）
- 配置文件分离两种令牌类型

### 2. 飞书文档内容填充 (a6b42293)
- 修复 `markdown_to_blocks` 函数使用正确的 block 格式
- 使用 `batchCreate` API 替代 `blocks/doc` API
- 支持标题 (heading)、列表 (bullet)、代码块 (code)、段落 (paragraph)
- 合并连续文本行为段落
- 修复 index_type 参数

### 3. 格式验证临时移除 (c57ca672)
- 临时移除 user_access_token 的 `u-` 开头格式验证
- 允许使用 tenant_access_token 作为 user_access_token
- 生产环境应使用真正的 user_access_token (u-开头)

## 问题根因

1. **令牌类型混淆**
   - 配置文件中的 `access_token` 实际是 tenant_access_token
   - 但代码把它当成 user_access_token 使用
   - 格式验证失败导致拒绝使用 token

2. **Block 格式错误**
   - 使用了旧的 `block_type` 数字格式 (1, 2, 3, 32)
   - 应该使用 `type` 字符串格式 (heading, paragraph, bullet, code)
   - 使用错误的 API 端点 (`blocks/doc`)
   - 应该使用 `batchCreate` 端点
