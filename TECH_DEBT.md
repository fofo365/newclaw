## 🔴 P0 - 阻塞性问题（需立即处理）

### 1. 缓存命中率低
**优先级**: P0  
**状态**: 待修复  
**发现时间**: 2026-03-10  
**影响范围**: 性能优化  

**问题描述**:
- 缓存命中率仅 **1.85%**，目标 **> 80%**
- 缓存键基于文本块而非完整文档
- 多块文档导致大量缓存未命中

**根本原因**:
```rust
// 当前实现：缓存键基于文本块
let cache_key = self.compute_cache_key(chunk);

// 问题：同一文档的不同块会产生不同的缓存键
// 导致：即使嵌入相同文档，也会因为分块不同而未命中
```

**影响**:
- 缓存效果不佳
- 性能提升未达预期
- API 调用成本增加

**解决方案**:
1. **短期**（1-2 天）：
   - 调整缓存策略：基于文档哈希而非块哈希
   - 实现文档级缓存

2. **长期**（v0.6.0）：
   - 实现智能缓存：语义相似度缓存
   - 分布式缓存：Redis 后端

**预计工作量**: 4-8 小时  
**负责人**: AI Agent (GLM-5)  
**截止日期**: 2026-03-11 (Week 1 Day 4)

## 🟡 P1 - 重要问题（本周内处理）

### 2. 长文本 Token 计数略慢
**优先级**: P1  
**状态**: 待优化  
**发现时间**: 2026-03-10  
**影响范围**: 性能  

**问题描述**:
- 10,000 字符文本 Token 计数需要 **121.27ms**
- 目标 **< 100ms**，超出 **21%**

**根本原因**:
- Tiktoken 库性能限制
- 无缓存机制

**解决方案**:
1. 缓存 Token 计数结果（优先级最高）
2. 并行计算（多线程）
3. 预估 Token 数（精度略降）

**预计工作量**: 2-4 小时  
**截止日期**: 2026-03-12 (Week 1 Day 5)
